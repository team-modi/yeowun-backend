# 전시 목록·상세·필터 강화 + 공공데이터 배치 시딩 설계 (스펙 ②)

- 상태: **계약 실호출 검증 완료(2026-07-02)** — 구현 계획(writing-plans) 대기
- 관련: 스펙 ①(`2026-07-02-전시-방문기록-스냅샷-design.md`), `docs/04_전시_구현.md`

## 1. 배경 · 목표

프론트 목업(목록 카드 / 필터 / 상세 3종)에 맞춰 실시간 전시 기능을 강화한다. 동시에 데이터 소스를
**배치 시딩(수집-적재)** 방식으로 확정한다(앞선 B안=매 조회 실시간 프록시는 폐기 — §11).

**목표**
- 공공데이터를 **1시간 주기 + 서버 부팅 시** DB에 시딩(upsert). 조회는 DB에서.
- 목록 카드: 포스터·제목·장소·지역·기간. 정렬 3종(최신순/종료순/인기순). 지역 필터. 기본=현재 진행중.
- 상세: 소개·관람료·주소·이미지·문의. "기록하기"로 방문기록(스펙 ①) 연결.

**비목표**
- 작가·세부 태그·12종 세부 장르 필터 — 공공 API 미제공(§3). 이번 범위에서 제외.
- `records` 도메인 컨벤션 정리(범위 밖, 스펙 ① 참조).

## 2. 데이터 소스 결정

**소스: 한눈에보는문화정보 (data.go.kr `15138937`)** — `apis.data.go.kr/B553457/cultureinfo`
- **목록**: `/realm2` (분야별). 전시만 → `realmCode=D000`. 페이지 파라미터 `PageNo`(대문자 P)·`numOfrows`(소문자 r).
- **상세**: `/detail2?seq={목록 seq}` → 소개·요금·주소·이미지 추가.
- 서버사이드 필터 지원: `sido`(지역), `from`/`to`(기간), `keyword`, `sortStdr`(1등록일·2명·3지역).

> ⚠️ 검토 이력: `api.kcisa.kr/openapi/API_CCA_145`(전시정보 통합, 24필드·작가 있음)도 실호출 확인했으나,
> **지역 필드·서버 필터가 없고** 데이터가 국립박물관 편중이라 필터 UX에 부적합. 사용자 결정으로 15138937 채택.
> 12종 세부 장르는 두 API 모두 미제공(15138937 realm은 "전시" 최상위 하나).

## 3. 확보 / 미확보 필드 (현실)

| 목업 필드 | 소스 | 처리 |
|---|---|---|
| 제목·기간(startDate/endDate 분리)·장소·지역(area/sigungu)·썸네일·좌표 | `/realm2` 목록 | ✅ |
| 소개(`contents1`)·관람료(`price`)·**주소(`placeAddr`, 도로명)**·이미지(`imgUrl`)·문의(`phone`)·URL | `/detail2` | ✅ (지연 조회 §7. contents1·price는 채움 변동) |
| **작가** | 없음 | ❌ 응답에서 제외(스키마 미포함) |
| **세부 태그/키워드** | 없음 | ❌ 제외 |
| **12종 세부 장르 필터** | 없음 | ❌ 지역·정렬만. 장르 칩은 이번 미구현(§10) |
| **운영시간** | 없음 | ❌ 제외 |
| **조회수(인기)** | 없음(소스 미제공) | ⚙️ **우리 앱 자체 조회수**로 대체(§9) |

> 검증(2026-07-02 실호출): `realmCode=D000` 전시 **266건**. 목록 `area`=시/도(부산·경기…), `sigungu`=시군구.
> 날짜는 **YYYYMMDD**(예 20180616). `detail2`는 `placeAddr`에 **도로명 주소**(예 "부산광역시 사하구 낙동남로 1191") 제공.

원칙(사용자 확정): **없는 필드는 억지로 안 채우고 null. 응답 필드셋은 항상 고정(일관).**

## 4. 아키텍처 · 데이터 흐름

```
[배치] @Scheduled(매시) + ApplicationRunner(부팅 시 1회)
   └─ ExhibitionFacade.syncCatalog()
        └─ /realm2?realmCode=D000 페이지 순회 → seq 기준 DB upsert (목록 12필드)

[조회 목록] GET /exhibitions
   └─ DB 쿼리(QueryDSL/Spec): 진행중 기본 + 지역 필터 + 정렬(최신/종료/인기) + 페이지네이션

[조회 상세] GET /exhibitions/{id}
   └─ DB에서 base 로드 → 상세필드(소개·요금·주소·이미지) 미캐시면 /detail2 1회 호출해 저장(지연 enrich §7)
   └─ our_view_count += 1
```

조회는 DB만 본다(외부 실시간 호출 없음). 상세필드만 최초 1회 지연 수집.

## 5. 스키마 변경 (V7 마이그레이션)

`exhibitions`에 컬럼 추가(기존 CATALOG/CUSTOM 통합 테이블 유지):
```sql
alter table exhibitions
    add column sigungu        varchar(50)  null,   -- 시군구(지역 세분)
    add column description    text         null,   -- detail2 contents1(소개)
    add column place_addr     varchar(500) null,   -- detail2 placeAddr(주소)
    add column img_url        varchar(2048) null,  -- detail2 imgUrl(고해상 이미지)
    add column phone          varchar(100) null,   -- detail2 phone(문의)
    add column place_url      varchar(2048) null,  -- detail2 placeUrl
    add column detail_synced_at datetime(6) null,  -- 상세 지연수집 완료 시각(null=미수집)
    add column our_view_count bigint not null default 0;  -- 우리 앱 조회수(인기순)
create index idx_exhibitions_view_count on exhibitions (our_view_count);
create index idx_exhibitions_period on exhibitions (start_date, end_date);
```
> 기존 컬럼 재사용: `price`(관람료), `poster_url`(썸네일/이미지), `region`(area 매핑), `external_id`(seq).
> `operating_hours`는 소스 미제공 → 항상 null(제거하지 않고 유휴).

## 6. 배치 동기화

- `ExhibitionFacade.syncCatalog()`를 `/realm2?realmCode=D000` 기반으로 재작성(현재 코드의 응답 파서를
  15138937 목록 스키마 `response.header.resultCode` + `body.items.item[]`에 맞게 정정 — §12 검증 후 확정).
- 페이지 순회(`PageNo`/`numOfrows`), `seq`(externalId) 기준 upsert. 종료 전시도 삭제하지 않음(누적).
- **스케줄러**: `@Scheduled(cron = "0 0 * * * *")`(매시 정각) + `@EnableScheduling`.
- **부팅 시딩**: `ApplicationRunner`가 기동 시 1회 `syncCatalog()`(cold start 방지). 기존 데모 시더와 통합/대체.
- 키 미설정·외부 장애: 로그만, 앱은 계속(기존 패턴). 데모용 샘플 시드는 유지(로컬 시연).

## 7. 상세 지연 수집 (detail2)

- `/detail2`는 일일 트래픽 제한(10000)·건별 호출이라 **전량 배치 금지**. 사용자가 상세를 열 때 **지연 수집 후 캐시**.
- `getDetail`: DB base 로드 → `detail_synced_at`이 null이면 `/detail2?seq=externalId` 호출 →
  소개·요금·주소·이미지·문의 저장 + `detail_synced_at` 기록 → 응답. 이후엔 DB 캐시만.
- 외부 실패 시: base 필드만으로 응답(상세필드 null), `detail_synced_at`은 미기록(다음에 재시도).

## 8. 목록 API (`GET /api/v1/exhibitions`)

- 필터: `region`(지역), `date`(그 날짜 진행중 / 미지정 시 오늘=진행중 기본), `keyword`. **`category`(장르) 파라미터는 이번 제외**(§10).
- 정렬 `sort`: `latest`(최신순=startDate DESC 기본), `ending`(종료순=endDate ASC), `popular`(인기순=our_view_count DESC, 동점 시 startDate DESC).
- 페이지네이션: 기존 `page`/`size`. 전부 **DB 쿼리(QueryDSL/Spec)** 로 처리(인메모리 아님).
- 카드 응답 필드: `exhibitionId`, `type`, `title`, `posterUrl`, `startDate`, `endDate`, `place`, `region`. (작가 없음)

## 9. 조회수 · 인기순

- 소스에 조회수 없음 → **우리 앱 자체 조회수(`our_view_count`)** 사용.
- `getDetail` 호출 시 해당 전시 `our_view_count += 1`(Entity 메서드 `increaseView()`).
- 인기순 = `our_view_count DESC`. ⚠️ 콜드스타트: 초기 전부 0 → 누적 전엔 사실상 최신순(동점 tiebreak)과 유사.
  (참고: 포털 조회수 합산안은 이 API가 조회수를 안 줘서 불가 — 스펙 ①·② 논의 확정.)

## 10. enum 확장 · 장르 필터

- **지역(region)**: `/realm2`·목록의 `area`(시/도 텍스트)를 enum으로 매핑. 목업 셋으로 확장:
  SEOUL·GYEONGGI·INCHEON·DAEGU·GYEONGBUK·BUSAN·ULSAN·GYEONGNAM·SEJONG·JEONNAM·JEONBUK·JEJU·CHUNGNAM·CHUNGBUK(+ETC).
  `fromAreaText`를 이 셋에 맞게 재작성(매칭 실패=ETC). 서버 필터는 `/area2` `sido` 또는 DB `region` 컬럼으로.
- **장르(category)**: 소스가 12종 세부 장르(드로잉/회화/사진…)를 **미제공** → **이번 릴리스 제외**.
  UI 장르 칩은 데이터 확보 전까지 비활성/미노출 권장. (향후: 별도 분류 소스나 수동 태깅 필요)

## 11. B 리팩토링 폐기 · A안 복귀

- 앞서 커밋 전 상태로 작업한 B(실시간 프록시) 변경 폐기:
  `ExhibitionFacade.search/getDetail`(인메모리), `CatalogExhibitionData.catalogId()`, `ExhibitionResult.fromCatalog`,
  `findAllCustomByOwner`(목록 병합용) 등 → DB 기반 조회로 복귀(원래 A안 구조 + 본 스펙 확장).
- 리팩토링 브랜치 산출물 중 재사용 가능한 것만 남기고 나머지는 되돌린다.

## 12. 설정 · 키 (✅ 실호출 검증 완료 2026-07-02)

- `base-url`: `https://apis.data.go.kr/B553457/cultureinfo` (기존 `http://api.kcisa.kr/...API_CCA_145`에서 교체, **https 필수**).
- 키: **data.go.kr 15138937 일반 인증키(64자)**. `.env`의 `CULTURE_API_KEY`를 이 값으로 교체 완료(이전엔 다른 KCISA 키가 들어가 있었음).
- **검증된 계약**:
  - 성공: `<response><header><resultCode>00</resultCode><resultMsg>정상입니다.</resultMsg></header><body>…`. 응답 **XML**.
  - 목록 `/realm2?realmCode=D000&PageNo=1&numOfrows=N` → `body.totalCount`(266) + `items.item[]`.
  - item(목록): `seq,title,startDate,endDate,place,realmName,area,sigungu,thumbnail,gpsX,gpsY,serviceName`. 날짜 **YYYYMMDD**.
  - `/detail2?seq=` → 위 + `price,contents1,url,phone,imgUrl,placeUrl,placeAddr,placeSeq`.
  - 파라미터 대소문자 주의: `PageNo`(대문자 P)·`numOfrows`(소문자 r)·`realmCode`·`serviceKey`.
- ⚠️ 이 키는 Discord 이미지로 노출됨 → 필요 시 data.go.kr에서 재발급 권장.

## 13. 테스트

- **동기화(단위/통합)**: `/realm2` 목 응답 → upsert 적재, 재동기화 멱등.
- **목록(통합)**: 진행중 기본, 지역 필터, 정렬 3종(최신/종료/인기), 페이지네이션.
- **상세 지연수집**: `detail_synced_at` null→호출·저장, 캐시 후 미호출, 실패 시 base만.
- **조회수**: 상세 조회마다 +1, 인기순 반영.
- **스케줄러/부팅**: 부팅 시 1회 시딩 실행(빈 DB 방지).
