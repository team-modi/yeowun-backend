# (시스템) 카탈로그 동기화

> 시나리오 2.10 — 시스템이 외부 공공데이터 전시 API를 수집해 DB에 적재한다. 부팅 직후 1회(cold start 방지), 이후 매일 자정에 재동기화하며 상세 보강도 이때 함께 수행한다.

**다이어그램이 필요한 이유**
- 조건 분기: 인증키 미설정 스킵, 기간 비정상 레코드 스킵, externalId 기준 upsert(갱신/신규)
- 데이터 보존 규칙: 재적재 시 상세 전용 필드(가격·설명·운영시간)는 건드리지 않는다 — 백필로 채운 값을 정기 동기화가 덮지 않게
- 실패 격리: 부적합 단건이 배치 전체를 중단시키지 않는다

```mermaid
sequenceDiagram
    actor 스케줄러
    participant ExhibitionFacade
    participant CultureExhibitionClient
    participant ExhibitionRepository
    participant Exhibition

    Note right of 스케줄러: 부팅 1회 ExhibitionCatalogBootSync / 매일 자정 ExhibitionSyncScheduler.syncDaily

    스케줄러->>ExhibitionFacade: 카탈로그 동기화
    ExhibitionFacade->>CultureExhibitionClient: 전체 목록 수집 (페이지 순회)
    alt 인증키 미설정
        CultureExhibitionClient-->>스케줄러: 외부 호출 없이 0건 종료
    end
    alt 외부 API 통신 실패
        CultureExhibitionClient-->>스케줄러: 실패 (경고 로그 후 다음 주기 재시도)
    end

    loop 수집 데이터 건별
        alt 기간 비정상 (종료일 < 시작일)
            ExhibitionFacade->>ExhibitionFacade: 스킵 (배치는 계속)
        else 정상
            ExhibitionFacade->>ExhibitionRepository: externalId로 기존 행 조회
            alt 기존 행 존재
                ExhibitionFacade->>Exhibition: 목록 필드 갱신
                Note right of Exhibition: 상세 전용 필드(가격·설명·운영시간)와 detailSyncedAt은 보존
            else 신규
                ExhibitionFacade->>Exhibition: CATALOG 전시 생성
            end
            ExhibitionFacade->>ExhibitionRepository: 저장
        end
    end

    ExhibitionFacade-->>스케줄러: 적재/갱신 건수 반환
```
