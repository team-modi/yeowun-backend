# 전시 목록/탐색

> 시나리오 2.3 — 누구나 로그인 없이 전시 목록을 탐색한다. 필터가 없으면 오늘 진행 중인 전시를 기본 노출하고, 로그인 시 본인의 CUSTOM 전시와 북마크 여부가 함께 반영된다.

**다이어그램이 필요한 이유**
- 조건 분기: 검색어 최소 2글자, 커서-정렬 판별자 일치, 거리순의 lat·lng 필수
- 정렬 경로 이원화: latest·ending·popular은 DB 키셋 페이지네이션, distance는 앱 레이어 정렬(best-effort)
- 도메인 간 협력: 목록 개인화를 위해 bookmark 도메인(ExhibitionBookmarkRepository)을 함께 조회

```mermaid
sequenceDiagram
    actor 사용자
    participant ExhibitionV1Controller
    participant ExhibitionFacade
    participant ExhibitionRepository
    participant ExhibitionBookmarkRepository

    Note right of 사용자: 선택 인증(비로그인 허용)

    사용자->>ExhibitionV1Controller: 전시 목록 조회 요청
    alt 날짜 형식 오류 (YYYY-MM-DD 아님)
        ExhibitionV1Controller-->>사용자: 실패
    end
    ExhibitionV1Controller->>ExhibitionFacade: 목록/탐색

    ExhibitionFacade->>ExhibitionFacade: 필터 정규화 (검색어·지역·카테고리·섹션·정렬·페이지 크기)
    alt 검색어가 2글자 미만
        ExhibitionFacade-->>사용자: 실패
    end
    Note right of ExhibitionFacade: 필터 없음 → 오늘 진행 중 전시 기본 노출

    alt 거리순 정렬 (sort=distance)
        alt lat·lng 미전달
            ExhibitionFacade-->>사용자: 실패
        end
        ExhibitionFacade->>ExhibitionRepository: 필터에 맞는 후보 전체 조회
        ExhibitionFacade->>ExhibitionFacade: 제곱거리 정렬 후 커서 위치부터 슬라이스
        Note right of ExhibitionFacade: 좌표 없는 전시는 뒤로
    else DB 키셋 정렬 (latest·ending·popular)
        ExhibitionFacade->>ExhibitionFacade: 커서 디코드
        alt 커서의 정렬 판별자가 현재 sort와 불일치
            ExhibitionFacade-->>사용자: 실패
        end
        ExhibitionFacade->>ExhibitionRepository: 키셋 한 페이지 조회 (size+1건)
        ExhibitionFacade->>ExhibitionRepository: 필터 전체 건수 조회
    end
    Note right of ExhibitionRepository: 비로그인은 CATALOG만, 로그인 시 본인 CUSTOM 포함

    ExhibitionFacade->>ExhibitionBookmarkRepository: 페이지 내 전시들의 북마크 여부 일괄 조회
    Note right of ExhibitionBookmarkRepository: 비로그인은 조회 없이 전부 false

    ExhibitionFacade-->>ExhibitionV1Controller: 결과 반환 (nextCursor·hasNext·totalCount)
    ExhibitionV1Controller-->>사용자: 성공
```
