# (시스템) 상세 백필

> 시나리오 2.10 — 시스템이 상세(가격·설명 등)를 아직 채우지 않은 CATALOG 전시를 행 단위로 수집해 보강한다. 상세 진입 없이도 무료 판정 등이 동작하게 하는 선반영 경로다.

**다이어그램이 필요한 이유**
- 트랜잭션 설계: 대상 조회와 행 단위 수집을 분리 — 다건 외부 호출을 한 트랜잭션에 오래 물지 않는다
- 조건 분기: 인증키 미설정 스킵, 원천에 상세 없음(확인 완료 표기 — 값은 안 채우고 재조회만 차단), 외부 실패(다음 주기 재시도)
- 실패 격리: 한 행이 실패해도 나머지 행은 계속 처리한다

```mermaid
sequenceDiagram
    actor 스케줄러
    participant CatalogEnricher
    participant ExhibitionFacade
    participant ExhibitionRepository
    participant CultureExhibitionClient
    participant Exhibition

    Note right of 스케줄러: 매시 정각 + 부팅 15초 후 10분 간격 (ExhibitionSyncScheduler)

    스케줄러->>CatalogEnricher: 상세 백필
    alt 원천 인증키 미설정
        CatalogEnricher-->>스케줄러: 스킵 (0건 — 상세 없음 오인 표기 방지)
    end

    CatalogEnricher->>ExhibitionFacade: 상세 미수집 CATALOG id 목록 조회
    ExhibitionFacade->>ExhibitionRepository: detailSyncedAt이 없는 CATALOG 조회 (실행당 상한)

    loop id별 (행 단위 짧은 트랜잭션)
        CatalogEnricher->>ExhibitionFacade: 상세 1건 수집
        ExhibitionFacade->>ExhibitionRepository: 전시 조회
        alt 없음 · CATALOG 아님 · 이미 상세 수집됨
            ExhibitionFacade-->>CatalogEnricher: 건너뜀
        end

        ExhibitionFacade->>CultureExhibitionClient: 단건 상세 조회
        alt 상세 수신
            ExhibitionFacade->>Exhibition: 상세 필드 반영 + 동기화 시각 기록
            ExhibitionFacade->>ExhibitionRepository: 저장
        else 원천에 상세 없음
            ExhibitionFacade->>Exhibition: 확인 완료 표기 (값은 채우지 않음)
            ExhibitionFacade->>ExhibitionRepository: 저장
            Note right of Exhibition: 재조회 대상에서 제외 — 매 주기 반복 호출 방지
        else 외부 호출 실패 (예외)
            ExhibitionFacade-->>CatalogEnricher: 실패 (경고 로그 후 다음 행 계속 — 다음 주기 재시도)
        end
    end

    CatalogEnricher-->>스케줄러: 상세를 채운 건수 반환
```
