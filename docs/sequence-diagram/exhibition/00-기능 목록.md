# exhibition 도메인 — 기능 목록

> 전시(Exhibition) 도메인의 API 기능과 시스템 유스케이스 전체. 상세 흐름은 각 시퀀스 다이어그램 문서를 참조한다.

| # | 기능명 | HTTP / 트리거 | 인증 | 한 줄 설명 | 다이어그램 |
|---|---|---|---|---|---|
| 01 | 전시 목록/탐색 | GET /api/v1/exhibitions | 선택 인증 | 검색어·섹션·지역·카테고리·날짜 필터 + 4가지 정렬(latest·ending·popular·distance) + 커서 페이지네이션 | [01-전시-목록-탐색.md](01-전시-목록-탐색.md) |
| 02 | 홈 배너 조회 | GET /api/v1/exhibitions/banners | 불필요 | 오늘 진행 중인 CATALOG 전시 중 조회수 상위 최대 3개 | [02-홈-배너.md](02-홈-배너.md) |
| 03 | 전시 상세 조회 | GET /api/v1/exhibitions/{exhibitionId} | 선택 인증 | 상세 반환 + 조회수 증가. 타인 CUSTOM은 403, CATALOG 최초 진입 시 외부 상세 1회 지연수집 | [03-전시-상세.md](03-전시-상세.md) |
| 04 | 개인 전시 등록 | POST /api/v1/exhibitions/custom | 필수 | CUSTOM 전시 등록. venueId로 장소·지역 파생, 분류기(랜덤/AI)가 장르 자동 부여 | [04-개인-전시-등록.md](04-개인-전시-등록.md) |
| 05 | (시스템) 카탈로그 동기화 | 부팅 1회 + 매시 정각 | — | 외부 공공데이터 전시 API 수집 → externalId 기준 upsert. 기간 비정상 레코드는 스킵 | [05-카탈로그-동기화.md](05-카탈로그-동기화.md) |
| 06 | (시스템) 장르 백필 | 매시 정각 + 부팅 15초 후 10분 간격 | — | 미분류 CATALOG 전시를 배치당 AI 1회 호출로 일괄 분류. 429 등 실패 시 랜덤 폴백 | [06-장르-백필.md](06-장르-백필.md) |
| 07 | (시스템) 상세 백필 | 매시 정각 + 부팅 15초 후 10분 간격 | — | 상세 미수집 CATALOG를 행 단위 짧은 트랜잭션으로 수집. 원천에 상세가 없으면 확인 완료 표기 | [07-상세-백필.md](07-상세-백필.md) |

**참고**

- 선택 인증: 비로그인 허용. 로그인 시 본인의 CUSTOM 전시 노출·북마크 여부 등 개인화가 반영된다.
- `ExhibitionFacade.getForSnapshot`은 기록(record) 도메인이 전시 스냅샷을 읽기 위한 내부 조회 메서드로, 독립 API/트리거가 아니라 별도 다이어그램을 두지 않는다(조회수 증가·외부 수집·개인화 없이 DB 조회 + 접근 검증만 수행).
