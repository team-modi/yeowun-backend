# 개인 전시 등록

> 시나리오 2.4 — 사용자가 목록에 없는 전시를 직접 등록해 기록의 대상으로 삼는다. 등록된 CUSTOM 전시는 등록자 본인에게만 노출된다.

**다이어그램이 필요한 이유**
- 도메인 간 협력: venueId 지정 시 venue 도메인에서 장소명·지역을 파생
- 조건 분기: 없는 venueId(404), 도메인 불변식 위반(제목·기간·개인전 작가)
- 부가 기능 격리: 장르 분류기는 실패해도 예외를 던지지 않아 등록 흐름을 깨지 않는다

```mermaid
sequenceDiagram
    actor 사용자
    participant ExhibitionV1Controller
    participant ExhibitionFacade
    participant VenueRepository
    participant GenreClassifier
    participant Exhibition
    participant ExhibitionRepository

    Note right of 사용자: 인증된 사용자

    사용자->>ExhibitionV1Controller: 개인 전시 등록 요청
    alt 날짜 형식 오류 (YYYY-MM-DD 아님)
        ExhibitionV1Controller-->>사용자: 실패
    end
    ExhibitionV1Controller->>ExhibitionFacade: 개인 전시 등록

    opt venueId 지정
        ExhibitionFacade->>VenueRepository: 전시관 조회
        alt 전시관이 존재하지 않음
            VenueRepository-->>사용자: 실패
        end
        ExhibitionFacade->>ExhibitionFacade: 장소명 파생 + (region 미지정 시) 지역 파생
    end

    ExhibitionFacade->>GenreClassifier: 장르 분류 (제목·카테고리·장소·작가 기반)
    Note right of GenreClassifier: 랜덤/AI 전략 — 실패해도 예외 없이 유효 장르 반환(폴백 보장)

    ExhibitionFacade->>Exhibition: CUSTOM 전시 생성
    alt 도메인 불변식 위반 (제목 1~100자 아님 · 시작일 > 종료일 · 개인전인데 작가 없음)
        Exhibition-->>사용자: 실패
    end

    ExhibitionFacade->>ExhibitionRepository: 저장

    ExhibitionFacade-->>ExhibitionV1Controller: 결과 반환
    ExhibitionV1Controller-->>사용자: 성공
```
