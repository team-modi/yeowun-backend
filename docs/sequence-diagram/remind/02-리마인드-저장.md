# 리마인드 저장

> 시나리오 2.8-2 — 사용자가 지금의 감정과 소감(오늘의 여운)을 남겨 리마인드를 저장한다. AI가 그때/지금의 감정 변화를 요약해 준다.

**다이어그램이 필요한 이유**
- 조건 분기: 원본 기록 존재 검증, 소유자 검증(403), AI 요약 SKIPPED/FAILED 분기
- 도메인 간 협력: Record 원본 조회 → AI 요약(AiChatClient) → Remind 생성·저장
- AI 요약은 **best-effort** — 미설정·쿨다운(SKIPPED)·오류(FAILED) 어떤 경우에도 저장 자체는 성공하며, AI 호출은 DB 트랜잭션 밖에서 수행한다
- 저장 시점에 원본 기록의 전시 카드 정보를 **스냅샷으로 복사**한다 — 원본이 삭제돼도 리마인드 카드는 유지된다

```mermaid
sequenceDiagram
    actor 사용자
    participant RemindV1Controller
    participant RemindFacade
    participant RecordJpaRepository
    participant RemindAiSummarizer
    participant AiRateLimiter
    participant AiChatClient
    participant Remind
    participant RemindJpaRepository

    Note right of 사용자: 인증된 사용자

    사용자->>RemindV1Controller: 리마인드 저장 요청
    RemindV1Controller->>RemindFacade: 리마인드 저장

    RemindFacade->>RecordJpaRepository: 원본 기록 조회 (감정 포함)
    alt 기록이 존재하지 않거나 삭제됨
        RecordJpaRepository-->>사용자: 실패
    end
    alt 본인 기록이 아님
        RemindFacade-->>사용자: 실패 (403)
    end

    Note over RemindFacade: 감정 코드 정규화 (공백 제거 · 중복 제거)

    RemindFacade->>RemindAiSummarizer: 감정 변화 요약 요청 (DB 트랜잭션 밖)
    RemindAiSummarizer->>AiRateLimiter: 호출 허용 확인
    alt AI 미설정 또는 쿨다운 중
        RemindAiSummarizer-->>RemindFacade: SKIPPED (요약 없음)
    else 호출 허용
        RemindAiSummarizer->>AiChatClient: 감정 변화 서술 생성
        alt 생성 실패 또는 빈 응답
            RemindAiSummarizer-->>RemindFacade: FAILED (요약 없음)
        else 생성 성공
            RemindAiSummarizer-->>RemindFacade: READY (요약, 최대 300자)
        end
    end
    Note over RemindFacade: 어떤 상태여도 저장은 진행 (best-effort)

    RemindFacade->>Remind: 리마인드 생성 (전시 스냅샷 복사 · 감정 · AI 요약 상태)
    alt 소감이 비어있음
        Remind-->>사용자: 실패
    end
    RemindFacade->>RemindJpaRepository: 리마인드 저장

    RemindFacade-->>RemindV1Controller: 결과 반환
    RemindV1Controller-->>사용자: 성공
```
