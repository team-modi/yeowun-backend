# 리마인드 저장

> 시나리오 2.8-2 — 사용자가 지금의 감정과 소감(오늘의 여운)을 남겨 리마인드를 저장한다. AI가 그때/지금의 감정 변화를 요약해 준다.

**다이어그램이 필요한 이유**
- 조건 분기: 원본 기록 존재 검증, 소유자 검증(403), AI 설정 여부(PENDING vs SKIPPED)
- 도메인 간 협력: Record 원본 조회 → Remind 생성·저장(즉시 응답) → **백그라운드 요약**(AiChatClient)
- AI 요약은 **응답을 막지 않는 백그라운드 best-effort**(M-2 — 지연 개선) — 저장 즉시 `PENDING`으로 응답하고, `aiExecutor`에서 요약을 만들어 Remind를 `READY`/`FAILED`로 갱신한다(미설정이면 `SKIPPED`로 즉시 확정). 이전에는 서블릿 스레드에서 동기 호출해 응답이 ~수 초 지연됐다.
- 저장 시점에 원본 기록의 전시 카드 정보를 **스냅샷으로 복사**한다 — 원본이 삭제돼도 리마인드 카드는 유지된다
- 백그라운드 작업은 **인메모리라 재시작·배포 시 유실**될 수 있다 → `RemindSummaryBackfillScheduler`가 주기적으로 PENDING을 훑어 복구하고, 포기 시간(기본 1일)을 넘긴 건은 FAILED로 확정해 **영구 PENDING을 없앤다**

```mermaid
sequenceDiagram
    actor 사용자
    participant RemindV1Controller
    participant RemindFacade
    participant RecordJpaRepository
    participant RemindAiSummarizer
    participant Remind
    participant RemindJpaRepository
    participant RemindSummaryBackfill
    participant aiExecutor
    participant AiChatClient

    Note right of 사용자: 인증된 사용자

    사용자->>RemindV1Controller: 리마인드 저장 요청
    RemindV1Controller->>RemindFacade: 리마인드 저장

    RemindFacade->>RecordJpaRepository: 원본 기록 조회 (감정 포함)
    alt 기록이 존재하지 않거나 삭제됨
        RecordJpaRepository-->>사용자: 실패
    end
    alt 본인 기록이 아님
        RemindFacade-->>사용자: 실패 (403)
    end

    Note over RemindFacade: 감정 코드 정규화 · AI 설정 여부로 초기 상태 결정
    RemindFacade->>RemindAiSummarizer: isEnabled()?
    alt AI 미설정
        RemindAiSummarizer-->>RemindFacade: false → SKIPPED로 확정
    else 설정됨
        RemindAiSummarizer-->>RemindFacade: true → PENDING
    end

    RemindFacade->>Remind: 리마인드 생성 (스냅샷 복사 · 감정 · 초기 AI 상태 · 요약 null)
    alt 소감이 비어있음
        Remind-->>사용자: 실패
    end
    RemindFacade->>RemindJpaRepository: 리마인드 저장 (커밋)
    opt 초기 상태 = PENDING
        RemindFacade->>RemindSummaryBackfill: 백그라운드 요약 예약 (remindId)
    end
    RemindFacade-->>RemindV1Controller: 결과 반환 (PENDING/SKIPPED · 요약 null)
    RemindV1Controller-->>사용자: 성공 (즉시 응답)

    Note over RemindSummaryBackfill,AiChatClient: 이하 커밋 후 백그라운드(aiExecutor) — 응답 이후 수행
    RemindSummaryBackfill->>aiExecutor: 실행 제출
    aiExecutor->>RemindAiSummarizer: 감정 변화 요약 (DB 트랜잭션 밖)
    RemindAiSummarizer->>AiChatClient: 감정 변화 서술 생성
    RemindAiSummarizer-->>aiExecutor: READY(요약) / FAILED / SKIPPED
    aiExecutor->>RemindJpaRepository: Remind 갱신 (요약·상태) — 짧은 tx
    Note over 사용자: 클라이언트는 GET 리마인드로 요약(READY)을 폴링

    Note over RemindSummaryBackfill,RemindJpaRepository: 재시작 등으로 위 백그라운드가 유실된 경우 — 백필 스케줄러가 복구
    RemindSummaryBackfill->>RemindJpaRepository: PENDING 조회 (유예 이전·오래된 순·batch 제한)
    alt 포기 시간(기본 1일) 초과
        RemindSummaryBackfill->>RemindJpaRepository: FAILED로 확정 (AI 호출 없음)
    else 재시도
        RemindSummaryBackfill->>RemindAiSummarizer: 요약 재생성
        Note over RemindSummaryBackfill: 쿨다운(SKIPPED)이면 PENDING 유지 — 다음 주기 재시도
    end
```
