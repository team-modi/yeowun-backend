# 리마인드 상세 조회

> 시나리오 2.8-4 — 사용자가 리마인드 상세(그때의 감상 vs 지금의 소감, AI 요약)를 조회한다.

**다이어그램이 필요한 이유**
- 조건 분기: 리마인드 존재 검증, 소유자 검증(403)
- 도메인 간 협력: "그때"(원본 기록 내용·감정)는 Record를 **라이브 조회** — 원본이 삭제됐으면 null/빈 목록으로 대체(실패 아님)

```mermaid
sequenceDiagram
    actor 사용자
    participant RemindV1Controller
    participant RemindFacade
    participant RemindJpaRepository
    participant RecordJpaRepository

    Note right of 사용자: 인증된 사용자

    사용자->>RemindV1Controller: 리마인드 상세 조회 요청
    RemindV1Controller->>RemindFacade: 상세 조회

    RemindFacade->>RemindJpaRepository: 리마인드 조회 (삭제 제외)
    alt 리마인드가 존재하지 않거나 삭제됨
        RemindJpaRepository-->>사용자: 실패
    end
    alt 본인 리마인드가 아님
        RemindFacade-->>사용자: 실패 (403)
    end

    RemindFacade->>RecordJpaRepository: 원본 기록 라이브 조회 (감정 포함)
    alt 원본 기록이 삭제됨
        Note over RemindFacade: 그때의 감상 null · 감정 빈 목록으로 대체 (실패 아님)
    end

    RemindFacade-->>RemindV1Controller: 결과 반환 (그때 vs 지금 · AI 요약)
    RemindV1Controller-->>사용자: 성공
```
