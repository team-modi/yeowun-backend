# remind 도메인 기능 목록

> 시나리오 2.8 — 사용자는 7일 이상 지난 기록을 다시 꺼내어, 지금의 감정과 소감을 남긴다.

| # | 기능명 | HTTP | 인증 | 설명 | 다이어그램 |
|---|---|---|---|---|---|
| 01 | 오늘의 소환 대상 조회 | `GET /api/v1/reminds/candidate` | 필요 | 작성 후 7일 이상 지났고 아직 회고하지 않은 내 기록 1건을 최신순으로 소환. 없으면 null(에러 아님). 전시 작가명은 라이브 조회 | [01-소환-대상-조회.md](01-소환-대상-조회.md) |
| 02 | 리마인드 저장 | `POST /api/v1/reminds` | 필요 | 원본 기록에 지금의 감정·소감을 남겨 저장. AI 감정 변화 요약은 best-effort(SKIPPED/FAILED여도 저장 성공), 전시 카드 정보는 스냅샷으로 복사 | [02-리마인드-저장.md](02-리마인드-저장.md) |
| 03 | 리마인드 목록 조회 | `GET /api/v1/reminds` | 필요 | 내 리마인드를 최신순 오프셋 페이징으로 조회. 소감은 60자 미리보기 | [03-리마인드-목록-조회.md](03-리마인드-목록-조회.md) |
| 04 | 리마인드 상세 조회 | `GET /api/v1/reminds/{remindId}` | 필요 | 그때의 감상(원본 기록, 라이브 조회) vs 지금의 소감 + AI 요약. 원본이 삭제됐으면 그때 정보는 null | [04-리마인드-상세-조회.md](04-리마인드-상세-조회.md) |
