# record 도메인 기능 목록

> 시나리오 2.6(기록), 2.7(AI 감상문)에 해당하는 기능 전체.
> 모든 기능은 로그인한 사용자만 사용할 수 있으며, 기록은 본인 것만 접근할 수 있다.

| # | 기능 | HTTP | 인증 | 설명 | 다이어그램 |
|---|---|---|---|---|---|
| 01 | 기록 작성 | `POST /api/v1/records` | 필요 | 관람일·감정·미디어 검증 후 전시 스냅샷을 박제해 기록을 저장한다 | [01-기록-작성.md](01-기록-작성.md) |
| 02 | 내 기록 목록 조회 | `GET /api/v1/records` | 필요 | 검색어·감정·전시·북마크·작성 방식·관람일 범위로 필터링, 오프셋 페이징 | [02-내-기록-목록-조회.md](02-내-기록-목록-조회.md) |
| 03 | 내가 다녀온 전시 목록 | `GET /api/v1/records/exhibitions/visited` | 필요 | 본인이 기록을 남긴 전시 목록 조회 (기록 목록과 동일한 검색 구현 재사용) | [03-내가-다녀온-전시-목록.md](03-내가-다녀온-전시-목록.md) |
| 04 | 기록 상세 조회 | `GET /api/v1/records/{recordId}` | 필요 | 본인 기록 상세 조회 (타인 기록은 403) | [04-기록-상세-조회.md](04-기록-상세-조회.md) |
| 05 | 기록 수정 | `PUT /api/v1/records/{recordId}` | 필요 | 감상·감정·미디어의 전체 교체, 레거시 키워드 정리 | [05-기록-수정.md](05-기록-수정.md) |
| 06 | 기록 삭제 | `DELETE /api/v1/records/{recordId}` | 필요 | 본인 기록의 논리 삭제 (soft delete) | [06-기록-삭제.md](06-기록-삭제.md) |
| 07 | 기록 북마크 | `POST /api/v1/records/{recordId}/bookmark` | 필요 | 본인 기록 북마크, 멱등 | [07-기록-북마크.md](07-기록-북마크.md) |
| 08 | 기록 북마크 해제 | `DELETE /api/v1/records/{recordId}/bookmark` | 필요 | 본인 기록 북마크 해제, 멱등 | [08-기록-북마크-해제.md](08-기록-북마크-해제.md) |
| 09 | AI 질문 생성 | `POST /api/v1/records/ai/questions` | 필요 | 전시 맥락 기반 질문 3개 생성 (쿨다운 429, 구조화 출력, aiExecutor 비동기) | [09-AI-질문-생성.md](09-AI-질문-생성.md) |
| 10 | AI 감상문 다듬기 | `POST /api/v1/records/ai/compose` | 필요 | 질문·답변을 1인칭 감상문(300자 이내)으로 다듬음 (저장하지 않음) | [10-AI-감상문-다듬기.md](10-AI-감상문-다듬기.md) |
