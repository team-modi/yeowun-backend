# AI 질문 생성

> 시나리오 2.7 — 사용자가 전시를 지정하면 AI가 그 전시 맥락에 맞는 질문 3개를 생성한다. "다른 질문 보기"는 같은 호출의 반복이다.

**다이어그램이 필요한 이유**
- 조건 분기: 사용자당 쿨다운(간격 내 재호출은 429) + 전시 접근 검증 + 생성 결과 검증
- 도메인 간 협력: 전시 상세를 도구처럼 조회해 프롬프트 맥락으로 만들고, LLM은 AiChatClient 포트로만 의존한다
- 비동기 실행: AI 호출은 전용 스레드풀(aiExecutor)에서 수행해 서블릿 워커 스레드를 빨리 반환한다
- 구조화 출력: 질문 3개 필드를 스키마로 강제해 자연어 JSON 파싱 없이 개수를 보장한다

```mermaid
sequenceDiagram
    actor 사용자
    participant RecordAiV1Controller
    participant RecordAiFacade
    participant AiRateLimiter
    participant ExhibitionFacade
    participant AiChatClient

    Note right of 사용자: 인증된 사용자

    사용자->>RecordAiV1Controller: AI 질문 생성 요청
    Note over RecordAiV1Controller: aiExecutor 스레드풀에서 비동기 실행
    RecordAiV1Controller->>RecordAiFacade: 질문 생성

    RecordAiFacade->>AiRateLimiter: 쿨다운 확인
    alt 최소 호출 간격이 지나지 않음
        AiRateLimiter-->>사용자: 실패 (429)
    end

    RecordAiFacade->>ExhibitionFacade: 전시 조회 (맥락용)
    alt 전시가 존재하지 않음
        ExhibitionFacade-->>사용자: 실패
    end
    alt 타인의 개인 전시
        ExhibitionFacade-->>사용자: 실패
    end

    Note over RecordAiFacade: 전시 맥락 프롬프트 구성 (설명 500자 제한·외부 데이터 가드 문구)
    RecordAiFacade->>AiChatClient: 구조화 출력 요청 (질문 3개 스키마)
    AiChatClient-->>RecordAiFacade: 질문 3개 반환

    Note over RecordAiFacade: 빈 질문 제거·공백 정리
    alt 유효한 질문이 하나도 없음
        RecordAiFacade-->>사용자: 실패
    end

    RecordAiFacade-->>RecordAiV1Controller: 질문 목록 반환
    RecordAiV1Controller-->>사용자: 성공 (저장하지 않음)
```
