# 기록 작성

> 시나리오 2.6 — 사용자가 전시를 선택해 감상(직접 작성 또는 AI 도움)·감정·관람일·사진/영상을 기록한다.

**다이어그램이 필요한 이유**
- 조건 분기: 관람일(미래 불가, 미입력 시 오늘 KST)·감정(1개 이상 필수)·미디어(최대 5개, 정렬 순서 중복 불가, 사진 10MB·영상 100MB) 검증
- 도메인 간 협력: Record가 Exhibition의 접근 가능 여부를 확인하고 스냅샷을 받아 박제한다
- 스냅샷 박제: 생성 시점의 전시 정보를 기록에 복사해, 이후 전시가 변경·삭제되어도 기록은 영향받지 않는다

```mermaid
sequenceDiagram
    actor 사용자
    participant RecordV1Controller
    participant RecordService
    participant ExhibitionFacade
    participant Record
    participant RecordJpaRepository

    Note right of 사용자: 인증된 사용자

    사용자->>RecordV1Controller: 기록 작성 요청
    RecordV1Controller->>RecordService: 기록 작성

    Note over RecordService: 관람일 결정 (미입력 시 오늘, KST)
    alt 관람일이 미래이거나 감정이 없음
        RecordService-->>사용자: 실패
    end
    alt 미디어 규칙 위반 (5개 초과·정렬 순서 중복·용량 초과)
        RecordService-->>사용자: 실패
    end

    RecordService->>ExhibitionFacade: 전시 조회 (스냅샷용)
    alt 전시가 존재하지 않음
        ExhibitionFacade-->>사용자: 실패
    end
    alt 타인의 개인 전시
        ExhibitionFacade-->>사용자: 실패
    end

    RecordService->>Record: 기록 생성 (전시 스냅샷 박제)
    RecordService->>Record: 감정·미디어 구성
    RecordService->>RecordJpaRepository: 저장

    RecordService-->>RecordV1Controller: 결과 반환
    RecordV1Controller-->>사용자: 성공
```
