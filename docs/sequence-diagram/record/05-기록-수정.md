# 기록 수정

> 시나리오 2.6 — 사용자가 자신의 기록을 수정한다. 감상·감정·미디어의 전체 교체 방식이다.

**다이어그램이 필요한 이유**
- 조건 분기: 기록 존재·소유자 검증 + 작성과 동일한 콘텐츠 규칙(관람일·감정·미디어) 검증
- 전체 교체: 부분 수정이 아니라 감상·감정·미디어를 통째로 갈아끼우고, 폐지된 키워드는 빈 목록으로 정리한다

```mermaid
sequenceDiagram
    actor 사용자
    participant RecordV1Controller
    participant RecordService
    participant RecordJpaRepository
    participant Record

    Note right of 사용자: 인증된 사용자

    사용자->>RecordV1Controller: 기록 수정 요청
    RecordV1Controller->>RecordService: 기록 수정

    RecordService->>RecordJpaRepository: 기록 조회 (삭제 제외)
    alt 기록이 존재하지 않거나 삭제됨
        RecordService-->>사용자: 실패
    end
    alt 본인 기록이 아님
        RecordService-->>사용자: 실패
    end

    Note over RecordService: 관람일 결정 (미입력 시 오늘, KST)
    alt 관람일이 미래이거나 감정이 없음
        RecordService-->>사용자: 실패
    end
    alt 미디어 규칙 위반 (5개 초과·정렬 순서 중복·용량 초과)
        RecordService-->>사용자: 실패
    end

    RecordService->>Record: 감상·관람일 교체
    RecordService->>Record: 감정 전체 교체
    RecordService->>Record: 키워드 정리 (키워드 개념 폐지 — 빈 목록으로)
    RecordService->>Record: 미디어 전체 교체
    RecordService->>RecordJpaRepository: 변경 반영 (flush)

    RecordService-->>RecordV1Controller: 수정된 상세 반환
    RecordV1Controller-->>사용자: 성공
```
