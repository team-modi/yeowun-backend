# 알림 목록 조회

> 시나리오 2.9 — 사용자가 자신의 알림 목록을 최신순으로 조회한다.

**다이어그램이 필요한 이유**
- 조건 분기: 커서 유무에 따른 첫 페이지/이후 페이지 분기 + 손상된 커서 검증(INVALID_CURSOR)
- 협력: Cursor가 opaque 토큰의 디코딩·정렬 일치 검증을 담당한다
- hasNext 판별: size+1개를 조회해 다음 페이지 존재 여부와 nextCursor를 계산한다

```mermaid
sequenceDiagram
    actor 사용자
    participant NotificationV1Controller
    participant NotificationFacade
    participant Cursor
    participant NotificationRepository

    Note right of 사용자: 인증된 사용자

    사용자->>NotificationV1Controller: 알림 목록 조회 요청 (cursor, size)
    NotificationV1Controller->>NotificationFacade: 알림 목록 조회

    NotificationFacade->>Cursor: 커서 디코딩 (정렬 판별자 latest)
    alt 커서가 손상되었거나 정렬 불일치
        Cursor-->>사용자: 실패 (INVALID_CURSOR)
    end

    NotificationFacade->>NotificationRepository: 페이지 조회 (size+1개, 최신순 키셋)
    alt 커서 없음
        NotificationRepository-->>NotificationFacade: 첫 페이지
    else 커서 있음
        NotificationRepository-->>NotificationFacade: 커서 이후 페이지
    end

    alt size+1개 조회됨 (다음 페이지 있음)
        NotificationFacade->>Cursor: 마지막 항목으로 nextCursor 인코딩
    else size 이하 (마지막 페이지)
        NotificationFacade->>NotificationFacade: nextCursor = null
    end

    NotificationFacade->>NotificationRepository: 전체 개수 조회

    NotificationFacade-->>NotificationV1Controller: 결과 반환
    NotificationV1Controller-->>사용자: 성공
```
