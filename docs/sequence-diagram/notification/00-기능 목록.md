# notification 도메인 기능 목록

> 시나리오 2.9 — 사용자는 홈 종 아이콘에서 알림 목록을 확인하고 읽음 처리한다.

| # | 기능명 | HTTP | 인증 | 한 줄 설명 | 다이어그램 |
|---|---|---|---|---|---|
| 01 | 알림 목록 조회 | `GET /api/v1/notifications` | 필요 | 내 알림을 최신순 키셋 커서 페이지네이션으로 조회한다. 손상된 커서는 INVALID_CURSOR(400). | [01-알림-목록-조회.md](01-알림-목록-조회.md) |
| 02 | 알림 읽음 처리 | `PUT /api/v1/notifications/{notificationId}/read` | 필요 | 내 알림 한 건을 읽음 처리한다(멱등). 없거나 타인 알림이면 NOTIFICATION_NOT_FOUND(404). | [02-알림-읽음-처리.md](02-알림-읽음-처리.md) |

**비고**

- 알림 생성(발송) 배치는 미구현 — 현재 API는 조회·읽음 처리만 제공하며, `Notification.create(...)`를 호출하는 발송 주체가 아직 없다.
