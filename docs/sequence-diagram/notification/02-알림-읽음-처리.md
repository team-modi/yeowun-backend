# 알림 읽음 처리

> 시나리오 2.9 — 사용자가 알림을 읽음 처리한다. 이미 읽은 알림을 다시 읽어도 결과는 동일하다(멱등).

**다이어그램이 필요한 이유**
- 조건 분기: 알림 존재 + 소유자 검증 — 없거나 타인 알림이면 404(존재를 노출하지 않음)
- 도메인 책임: 소유자 판정(isOwnedBy)과 읽음 상태 변경(markRead)은 Notification의 책임
- 멱등: 이미 읽음이어도 같은 결과로 성공한다

```mermaid
sequenceDiagram
    actor 사용자
    participant NotificationV1Controller
    participant NotificationFacade
    participant NotificationRepository
    participant Notification

    Note right of 사용자: 인증된 사용자

    사용자->>NotificationV1Controller: 알림 읽음 처리 요청
    NotificationV1Controller->>NotificationFacade: 알림 읽음 처리

    NotificationFacade->>NotificationRepository: 알림 조회
    NotificationFacade->>Notification: 소유자 확인
    alt 알림이 존재하지 않거나 타인의 알림
        NotificationFacade-->>사용자: 실패 (NOTIFICATION_NOT_FOUND, 404)
    end

    NotificationFacade->>Notification: 읽음 처리 (멱등)
    NotificationFacade->>NotificationRepository: 저장

    NotificationFacade-->>NotificationV1Controller: 결과 반환
    NotificationV1Controller-->>사용자: 성공
```
