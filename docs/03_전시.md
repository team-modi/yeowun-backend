# 3. 전시 도메인

> 공통 규칙은 [00_개요_및_공통규칙.md](./00_개요_및_공통규칙.md), 인증 방식·권한(인가) 정책은 [01_인증인가.md](./01_인증인가.md) 참조.


## 3.0 개요
- **한 줄 설명**: 전시를 탐색/검색/상세 조회하고, 카탈로그에 없는 개인 전시를 사용자가 직접 등록한다.
- **용어 정의**
  - `카탈로그 전시(CATALOG)`: 외부 공개 전시 API/수집 데이터로 제공되는 공용 전시. `⚠가정` 출처는 외부 공개 전시 API.
  - `개인 전시(CUSTOM)`: 사용자가 직접 입력해 등록한 전시. 등록자 본인 기록에만 쓰인다.
  - `진행 중인 전시`: `startDate ≤ 오늘 ≤ endDate`인 전시.
- **범위 밖**: 전시 좋아요/관심 저장(통합기획서 P2 "관심 전시 저장 P1" → Should/추후), 전시 공유(P1 → 추후), 전시 추천 알고리즘.
- **결정 사항**: 유저플로우 B "개인 전시 직접 등록"을 반영. 기록은 항상 하나의 `exhibitionId`(CATALOG 또는 CUSTOM)에 연결된다.
- **모델링 방식 미확정(통합기획서 안건11)**: 통합기획서의 저장 데이터 모델(7장)은 개인전을 별도 엔티티가 아니라 **기록(Record)에 내장된 `customExhibition`(name 필수+place+date)** 으로 다루고, `exhibitionId`는 선택값이 된다. 본 명세는 대신 **CUSTOM 전시를 독립 엔티티**(`exhibitions` 테이블의 `type=CUSTOM`)로 모델링해 등록·재사용·상세조회가 가능하게 했다. 통합기획서 안건11이 "담당: 김규나 협의, 미해결"로 명시되어 있으므로 **이 설계 선택은 확정이 아니다** — 두 안의 트레이드오프는 99_부록.md 부록 B를 참고.

## 3.1 API 목록
| METHOD | URI | 인증 필요 | 설명 |
| --- | --- | --- | --- |
| GET | `/api/v1/exhibitions` | X | 전시 목록/탐색/검색(필터·정렬·페이지) |
| GET | `/api/v1/exhibitions/{exhibitionId}` | X | 전시 상세 |
| POST | `/api/v1/exhibitions/custom` | O | 개인 전시 직접 등록 |

## 3.2 도메인 RULE
**RULE: 전시 기간**
- `startDate ≤ endDate` 여야 함. 위반 → `400 / INVALID_INPUT`.

**RULE: 지역/카테고리 enum** `⚠가정` (실제 값 팀 확정)
- `region`: `SEOUL` | `GYEONGGI` | `BUSAN` | `DAEGU` | `ETC` ...
- `category`: `PAINTING`(회화) | `PHOTO`(사진) | `MEDIA`(미디어) | `SCULPTURE`(조각) | `ETC` ...

## 3.3 API 상세

### 3.3.1 전시 목록/탐색 — `GET /api/v1/exhibitions`
- **인증 필요**: X
- **권한**: 공개. (단 `CUSTOM` 전시는 등록자 본인에게만 노출 — 비로그인/타인에겐 미포함)

**Request (query)**
| 필드 | 타입 | 필수 | 제약 | 예시 |
| --- | --- | --- | --- | --- |
| keyword | string | X | 전시명·전시장명·작가명 부분 일치(통합기획서 5-2 검색 대상) | `"모네"` |
| date | string | X | `YYYY-MM-DD`. 해당 날짜에 진행 중인 전시 | `"2026-06-30"` |
| region | string(enum) | X | `RULE: 지역` | `"SEOUL"` |
| category | string(enum) | X | `RULE: 카테고리` | `"PHOTO"` |
| page,size,sort | - | X | `RULE: 페이지네이션`. 기본 sort `⚠가정` `startDate,DESC` | |

**처리 로직**
1. 필터 미지정 시 기본은 **진행 중인 전시** 목록(랜딩 기본). (전시탐색 유저플로우 "진행 중인 전시 조회")
2. `keyword`/`date`/`region`/`category`로 필터링.
3. 페이지네이션 적용해 반환. 비로그인이면 CATALOG만, 로그인이면 CATALOG + 본인 CUSTOM.

**Success Response** — `200`
```json
{
  "meta": { "result": "SUCCESS" },
  "data": {
    "content": [
      { "exhibitionId": 51, "type": "CATALOG", "title": "모네: 빛의 정원",
        "posterUrl": "https://...", "startDate": "2026-06-01", "endDate": "2026-09-30",
        "place": "예술의전당", "region": "SEOUL", "category": "PAINTING" }
    ],
    "page": 0, "size": 20, "totalElements": 12, "totalPages": 1, "hasNext": false
  }
}
```

**Error Cases**
| 상황 | HTTP Status | errorCode |
| --- | --- | --- |
| 잘못된 enum/날짜 포맷 | 400 | `INVALID_INPUT` |
| 외부 전시 API 장애 | 503 | `EXTERNAL_API_UNAVAILABLE` |

**예외/엣지 케이스**
- **조건에 맞는 전시 없음** → `200` + 빈 배열(`content: []`). 404 아님. (유저플로우 "조건에 맞는 전시가 없습니다" UI 상태)
- 미래 날짜 검색 가능(예정 전시). 과거 날짜도 조회 가능.

**인수 조건**
- Given 필터 없음, When 조회, Then `200` + 오늘 진행 중인 전시 목록.
- Given 조건 불충족 날짜, When 조회, Then `200` + `content: []`.

### 3.3.2 전시 상세 — `GET /api/v1/exhibitions/{exhibitionId}`
- **인증 필요**: X (CUSTOM은 본인만)
- **권한**: CATALOG 공개 / CUSTOM은 등록자 본인만

**Request (path)**: `exhibitionId` (long, 필수)

**처리 로직**
1. 전시 조회.
2. CUSTOM이면 요청자가 등록자인지 확인.

**Success Response** — `200`
```json
{
  "meta": { "result": "SUCCESS" },
  "data": {
    "exhibitionId": 51, "type": "CATALOG", "title": "모네: 빛의 정원",
    "posterUrl": "https://...", "startDate": "2026-06-01", "endDate": "2026-09-30",
    "place": "예술의전당", "region": "SEOUL", "category": "PAINTING",
    "description": "전시 소개...", "operatingHours": "10:00-18:00(월요일 휴관)", "price": "성인 20,000원",
    "artists": ["클로드 모네"], "keywords": ["인상주의", "고요함"]
  }
}
```
- `operatingHours`·`price`·`artists`·`keywords`는 통합기획서 P2 화면 구성("운영 시간/관람료: 데이터 있을 경우", "작가/키워드: 참여 작가, 장르/분위기")에 따라 선택 데이터다. 없으면 `null` 또는 빈 배열로 반환.

**Error Cases**
| 상황 | HTTP Status | errorCode |
| --- | --- | --- |
| 존재하지 않는 전시 | 404 | `NOT_FOUND` |
| 타인의 CUSTOM 전시 접근 | 403 | `FORBIDDEN` |

**인수 조건**
- Given 존재하는 CATALOG 전시, When 비로그인 조회, Then `200`.
- Given 타인 CUSTOM 전시 ID, When 조회, Then `403 / FORBIDDEN`.

### 3.3.3 개인 전시 등록 — `POST /api/v1/exhibitions/custom`
- **인증 필요**: O
- **권한**: 본인(등록자)
- 멱등성: `RULE: 멱등성` 적용 가능.

**Request (body)**
| 필드 | 타입 | 필수 | 제약 | 예시 |
| --- | --- | --- | --- | --- |
| title | string | O | 1~100자 | `"동네 골목 사진전"` |
| place | string | X | ≤100자 | `"성수동 카페"` |
| startDate | string | X | `YYYY-MM-DD` | `"2026-06-20"` |
| endDate | string | X | `YYYY-MM-DD`, `RULE: 전시 기간` | `"2026-06-30"` |
| region | string(enum) | X | `RULE: 지역` | `"SEOUL"` |
| category | string(enum) | X | `RULE: 카테고리` | `"PHOTO"` |
| posterUrl | string | X | 업로드된 이미지 URL | `"https://..."` |

**처리 로직**
1. 입력 검증(제목 필수).
2. `type=CUSTOM`, `ownerId=요청자`로 전시 저장.
3. 생성된 `exhibitionId` 반환 → 곧바로 기록 작성에 연결.

**Success Response** — `201`
```json
{ "meta": { "result": "SUCCESS" }, "data": { "exhibitionId": 90122, "type": "CUSTOM" } }
```

**Error Cases**
| 상황 | HTTP Status | errorCode |
| --- | --- | --- |
| 제목 누락 | 400 | `INVALID_INPUT` |
| 종료일 < 시작일 | 400 | `INVALID_INPUT` |
| 미인증 | 401 | `UNAUTHORIZED` |

**예외/엣지 케이스**
- 같은 사용자가 동일 제목 전시 여러 개 등록 가능(중복 허용). `⚠가정`

**인수 조건**
- Given 로그인 사용자, When 제목만 입력해 등록, Then `201` + `exhibitionId` 반환.
- Given 비로그인, When 등록, Then `401`.
