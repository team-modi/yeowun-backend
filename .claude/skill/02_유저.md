# 2. 유저 도메인 (프로필 포함)

> 공통 규칙은 [00_개요_및_공통규칙.md](./00_개요_및_공통규칙.md), 인증 방식·권한 정책은 [01_인증인가.md](./01_인증인가.md) 참조. 취향 키워드는 [05_기록_아카이브.md](./05_기록_아카이브.md) 기록의 AI 키워드 집계값.

## 2.0 개요
- **한 줄 설명**: User 엔티티(닉네임·프로필 이미지·소셜 provider·연령대·거주지역)를 정의하고, 프로필 조회/수정과 AI 취향 키워드·활동 통계를 제공한다.
- **용어 정의**
  - `회원(User)`: `provider` + 소셜 고유 ID 조합으로 식별되는 우리 서비스 사용자. 최초 생성은 [01_인증인가.md](./01_인증인가.md)의 소셜 로그인 API에서 일어나지만, 필드 정의·검증 규칙의 소유권은 이 도메인에 있다.
  - `연령대(ageGroup)`: 정확한 나이 대신 받는 구간값(통합기획서 안건5 "정확한 나이 대신 연령대"). 10대~50대 이상 + 선택 안 함.
  - `거주지역(residenceRegion/residenceDistrict)`: 시/도 + 구/군 단위까지만 받는다(통합기획서 안건6 "시/도 + 구 단위", 상세 주소는 미수집).
- **범위 밖**: 깊은 취향 분석 확장(Could), 취향 리포트(Should), 회원 탈퇴, 계정 정지/제재, 실명 수집(통합기획서 안건4 "닉네임 필수, 실명 선택 또는 제외" → 본 명세는 실명 필드 자체를 두지 않음).
- **결정 사항(문서 충돌 해소)**:
  1. **가입 시 별도 입력 화면 여부**: 기능정의서 `M1-103 닉네임 설정(Must)` + 와이어프레임 "유저 정보 입력 페이지" + 통합기획서 P4("회원 정보 입력" Must, 닉네임·연령대·지역 수집) vs 유저플로우 26.06.29 "개인정보는 소셜 연동 시 받아오기로 하여 입력 플로우 삭제" → **유저플로우(가장 최근 수정)를 채택**: 별도 가입 입력 화면 없음. 닉네임은 소셜 프로필에서 자동 세팅.
  2. 다만 통합기획서가 명시한 **연령대·거주지역 수집 자체의 필요성**(아카이브·리마인드·취향 분석·지역 기반 추천에 활용)은 유효하므로, 가입 화면 삭제로 사라지지 않게 **프로필 수정 API의 선택 입력 필드**로 흡수했다. 즉 가입 직후엔 닉네임만 자동 세팅되고, 연령대·거주지역은 사용자가 원할 때 프로필에서 채워 넣는 구조다.

## 2.1 도메인 RULE
**RULE: 닉네임**
- 1~20자, 공백만으로 구성 불가.
- 소셜 닉네임이 없거나 중복일 경우 → `⚠가정` `여운user_{랜덤6자리}`로 자동 생성.
- 위반(프로필 수정 시) → `400 / INVALID_NICKNAME`.
- 이 RULE은 [01_인증인가.md](./01_인증인가.md)의 소셜 로그인(신규 회원 생성)과 본 도메인의 프로필 수정 양쪽에서 공통으로 적용된다.

**RULE: 연령대(ageGroup)** (통합기획서 안건5)
- enum: `TEENS`(10대) | `TWENTIES`(20대) | `THIRTIES`(30대) | `FORTIES`(40대) | `FIFTIES_PLUS`(50대 이상) | `UNSPECIFIED`(선택 안 함, 기본값)
- 선택 입력. 미입력 시 `UNSPECIFIED`로 저장.
- 위반(미정의 코드) → `400 / INVALID_INPUT`.

**RULE: 거주지역(residenceRegion / residenceDistrict)** (통합기획서 안건6)
- `residenceRegion`: 시/도 단위 enum. `⚠가정` 실제 값은 [03_전시.md](./03_전시.md)의 전시 `region` enum과 동일 코드 집합을 재사용할지, 별도 행정구역 enum(전국 17개 시/도)을 쓸지 팀 확정 필요 — 의미가 다르다(전시는 "전시가 열리는 지역", 유저는 "거주지역")는 점에 주의.
- `residenceDistrict`: 구/군 단위 자유 문자열(또는 enum, `⚠가정`). 상세 주소는 수집하지 않는다.
- 둘 다 선택 입력. `residenceDistrict`만 있고 `residenceRegion`이 없는 조합은 불가 → `400 / INVALID_INPUT`.

## 2.2 API 목록
| METHOD | URI | 인증 필요 | 설명 |
| --- | --- | --- | --- |
| GET | `/api/v1/users/me` | O | 프로필 + 취향 키워드 + 활동 통계 조회 |
| PATCH | `/api/v1/users/me/profile` | O | 프로필 수정(부분 갱신) |

> URI·메서드는 통합기획서 8장 API 초안(`GET /users/me`, `PATCH /users/me/profile`)을 따른다. 이전 버전의 `PUT /api/v1/users/me`(전체 교체 가정)에서 **PATCH로 변경** — 어차피 "전달된 필드만 갱신"하는 부분 수정 의미였고, 실제 기획 문서가 PATCH로 명시했으므로 그에 맞춤(부록 A 갱신).

## 2.3 API 상세

### 2.3.1 프로필 조회 — `GET /api/v1/users/me`
- **인증 필요**: O / **권한**: 본인

**처리 로직**
1. 본인 식별.
2. 프로필 + AI 취향 키워드(기록 키워드 집계) + 활동 통계(기록 수, 다녀온 전시 수 등) 반환.

**Success Response** — `200`
```json
{
  "meta": { "result": "SUCCESS" },
  "data": {
    "userId": 1024, "nickname": "여운러", "profileImageUrl": "https://...",
    "provider": "KAKAO",
    "ageGroup": "TWENTIES", "residenceRegion": "SEOUL", "residenceDistrict": "강남구",
    "tasteKeywords": ["따뜻한 색감", "정적인 몰입", "현대미술"],
    "stats": { "recordCount": 23, "exhibitionCount": 17, "bookmarkCount": 5 }
  }
}
```
- 취향 키워드 산출 방식 `⚠가정`: 본인 기록의 `aiKeywords` 빈도 상위 N개. (별도 추천 모델 아님)
- `ageGroup`이 `UNSPECIFIED`이거나 `residenceRegion`/`residenceDistrict`가 미입력이면 해당 필드는 `null`로 반환.

**Error Cases**: 미인증 `401`.

**인수 조건**
- Given 로그인 사용자, When 조회, Then `200` + 프로필/취향/통계.

---

### 2.3.2 프로필 수정 — `PATCH /api/v1/users/me/profile`
- **인증 필요**: O / **권한**: 본인
- **부분 갱신(PATCH)**: body에 포함된 필드만 갱신한다. 포함하지 않은 필드는 기존 값 유지.

**Request (body)**
| 필드 | 타입 | 필수 | 제약 | 예시 |
| --- | --- | --- | --- | --- |
| nickname | string | X | `RULE: 닉네임` | `"여운러2"` |
| profileImageUrl | string | X | 업로드된 이미지 URL | `"https://..."` |
| ageGroup | string(enum) | X | `RULE: 연령대` | `"TWENTIES"` |
| residenceRegion | string(enum) | X | `RULE: 거주지역`. `residenceDistrict`와 함께 입력 권장 | `"SEOUL"` |
| residenceDistrict | string | X | `RULE: 거주지역`. `residenceRegion` 없이 단독 입력 불가 | `"강남구"` |

**처리 로직**
1. 본인 식별.
2. 전달된 필드만 갱신. 닉네임·연령대·거주지역 각각 검증.

**Success Response** — `200` + 수정된 프로필(2.3.1과 동일 shape).

**Error Cases**
| 상황 | HTTP Status | errorCode |
| --- | --- | --- |
| 닉네임 규칙 위반 | 400 | `INVALID_NICKNAME` |
| 연령대 코드 오류 | 400 | `INVALID_INPUT` |
| residenceDistrict만 입력(region 없음) | 400 | `INVALID_INPUT` |
| 미인증 | 401 | `UNAUTHORIZED` |

**예외/엣지 케이스**
- 닉네임 전역 유일성 요구 여부 `⚠가정`: **유일성 불필요**(소셜 식별이 별도라 표시용). 필요 시 `409 / NICKNAME_DUPLICATED` 추가.
- `ageGroup: "UNSPECIFIED"`로 명시적 전달 시 → 이미 입력했던 연령대를 다시 비움(선택 안 함으로 되돌리기) 허용.

**인수 조건**
- Given 로그인 사용자, When 닉네임만 변경, Then `200` + 닉네임만 갱신, 나머지 필드 유지.
- Given 로그인 사용자, When `ageGroup`·`residenceRegion`·`residenceDistrict` 함께 입력, Then `200` + 모두 갱신.
- Given 빈 닉네임, When 변경, Then `400 / INVALID_NICKNAME`.
- Given `residenceDistrict`만 입력(`residenceRegion` 없음), When 변경, Then `400 / INVALID_INPUT`.

> 로그아웃은 [01_인증인가.md](./01_인증인가.md) 1.4.3 참조(프로필 화면의 로그아웃 버튼이 동일 API 호출).
